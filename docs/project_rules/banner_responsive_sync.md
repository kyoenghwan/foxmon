# 프로젝트 특수 지침: 배너(Banner) 동기화 규칙

## 📌 이 지침을 읽어야 하는 상황

AI는 다음 키워드 감지 시 이 파일을 **반드시 먼저** 읽는다:
**"배너", "banner", "광고", "가로 세로", "비율", "사이드 배너", "날개 배너"**

---

## 1. 핵심 원칙 (절대 준수)

> **"본문 메인 배너와 사이드 배너의 크기는 비례하며, 가로와 세로 길이만 서로 뒤바뀐 상태로 항상 동일한 비율(1.5 : 1)과 물리적 면적을 유지한다."**

### 1.1 비율 절대 거울 등식 (Golden Ratio Equality)
배너의 크기/비율 조정을 요청받을 경우, 두 타입의 배너 비율이 **역수** 관계로 일치하는지 반드시 확인해야 합니다.
- **메인(본문) 가로 배너**: `aspect-[3/2]` (폭 1.5 : 높이 1) - `premium-job-card.tsx`, `special-job-card.tsx`, `general-job-item.tsx`
- **사이드(날개) 세로 배너**: `aspect-[2/3]` (폭 1 : 높이 1.5) - `side-banners.tsx`

### 1.2 날개 배너의 폭(Width) 조작 계산법
CSS Grid 프레임워크 상, 화면이 줄어들면 컨테이너 폭에 비례해 본문 배너의 크기가 자동으로 작아집니다.
그러나 **사이드 배너는 `fixed` 속성이므로 메인 배너의 높이와 정확히 일치하는 px 값을 추적하여 날개 배너의 `Width`로 명시적 주입해야** 합니다.
1. Grid Cell Width 계산: `(컨테이너 폭 - 좌우 패딩 - (Gap 개수 * Gap 너비)) / 열(Cols) 갯수`
2. Grid Cell Height 계산: `Grid Cell Width / 1.5`
3. ⭐️ 이 계산된 **Grid Cell Height 값**을 사이드 배너의 **Width**(`w-[...px]`) 값으로 적용합니다.

**현재 기준 맵핑:**
- `xl` (1280px 뷰포트 / Container 920px) -> `xl:w-[132px]`
- `2xl` (1440px 뷰포트 / Container 1100px) -> `2xl:w-[127px]`
- `3xl` (1920px 뷰포트 / Container 1500px) -> `3xl:w-[148px]`
- `4xl` (2560px 뷰포트 / Container 2040px) -> `4xl:w-[208px]`

### 1.3 마진 클램핑 안전 장치
날개 배너가 본문 컨테이너 영역과 겹치면 치명적인 UI 버그(Overlap)가 발생합니다. 항상 중앙 기점 Offset과 컨테이너 1/2 지점 사이에 `15px`의 여백을 둡니다.

## 2. AI 필수 질문 프로토콜

배너 관련 레이아웃 수정 요청 시 반드시 아래 질문을 먼저 수행한다:

~~~text
⚠️ [프로젝트 특수 지침 적용: banner_responsive_sync_rules]

배너의 크기나 비율 변경을 요청하셨습니다.
본 프로젝트의 규칙에 따라 "메인 배너와 사이드 배너의 크기는 서로 비례(가로/세로 역수)"해야 합니다.

어느 범위까지 변경을 원하십니까?:
1) [내용/스타일만 수정] (비율/크기에는 손대지 않고 색상, 폰트만 변경)
2) [배너 비율 동기화 수정] (배너의 기준 비율을 변경하되 메인/사이드 배너 수식을 모두 재계산하여 동기화)
3) [부분 예외 수정] (예외적으로 모바일에서만 크기를 변경하는 경우)

선택: (1/2/3)
~~~

**옵션별 처리 방식:**
- **옵션 1 (내용/스타일만 수정)**: 기존 `aspect-` 비율 클래스를 절대 유지한 채 내부 내용만 수정합니다.
- **옵션 2 (배너 비율 동기화 수정)**: 새로운 비율(예: 2:1)이 요구되면 `00_master_router.md`의 구조계산법에 따라 컨테이너 Max-Width와 오프셋, Wing Width를 전면 재계산하는 **YAML 설계서**를 제시해야 합니다.
- **옵션 3 (부분 예외 수정)**: 해당 특정 해상도에서만 `[vw]` 또는 % 단위로 처리하되, 사이드 배너가 숨겨지는 구간인지 먼저 확인합니다.

## 3. 금지 사항 (Never Do)

~~~text
❌ 절대 금지 목록:
- 메인 배너의 비율(`aspect-[]`)을 수정하면서 사이드 배너를 수정하지 않는 행위
- 사이즈 계산 없이 사이드 배너에 임의의 고정 Width(예: `w-[150px]`)를 넣는 행위
- 사이드 배너의 간섭 여부 계산 없이 `tailwind.config.ts`의 `container` 폭을 넓히는 행위
~~~
